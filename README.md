# 🔩 Snap Frame Garage

**実在するサーボやセンサを組み合わせて自分だけのロボットを設計し、URDF / MJCF で書き出せる、ブラウザで動くロボット工作アプリ。**

子どもやライトユーザーが、インストール・アカウントなしで「組む → 整える → 書き出す → (姉妹シミュレータで)操縦する → 直す」のループを回せることを目指した MVP 実装です。企画の詳細は [docs/robot_builder_kikakusho.md](docs/robot_builder_kikakusho.md)(別紙:[フレーム方式検討](docs/frame_design_kentou.md) / [パーツカタログ仕様](docs/frame_parts_catalog.md))を参照。

座標系はROS REP-103に合わせ、**+Xが正面、+Yが左、+Zが上**です。3Dビュー右上のビューキューブから正面・側面・上面へ移動できます。

## 動かし方

```bash
npm install
npm run dev      # 開発サーバ(表示されたURLをブラウザで開く)
npm run build    # 本番ビルド(dist/ に静的ファイル一式)
npm test         # コアロジックの単体テスト(vitest)
```

静的ホスティングにそのまま置けます(サーバレス・ローカルファースト。作業中モデルはlocalStorageに自動保存)。

## はじめてのかたへ

初回起動時に、アプリの遊びかた(穴とピンの文法・バランス・パワーボックス)を実画面のスクリーンショットと図解で説明する**スライド式ポップアップ**が開きます(ヘッダの「❓せつめい」でいつでも再表示)。最後のスライドから「ひながた」「ガイドつき」「じゆうにつくる」のどれでも始められます。

## ひながた(テンプレート)

ヘッダの「🤖 ひながた」から、完成済みロボットを読み込んで改造スタートできます:
**🚗にりんしゃ / 🐕いぬがた4そく(8サーボ=S箱ちょうど) / 🤖ひとがた2そく(ぶらぶらの腕つき) / 🐜むしがた6そく / 🦕ヤンセン4足＋補助輪**。
ヤンセン4足版は左右2モータの機構に前後キャスターを加え、60部品で組みやすさと接地安定性を両立します。各駆動軸は本体から伸ばしたL字アングルと側板で固定され、回転するクランクから独立しています。
すべて通常の穴ピン接続だけで組まれており(`src/data/templates.ts`)、手組みと同じデータなので自由にばらして改造できます。
テンプレート実装で見えたパーツカタログの過不足は [docs/template_parts_memo.md](docs/template_parts_memo.md) に整理しています。

## 遊び方(コアループ)

1. **組む** — 左のカタログからパーツを選ぶと**半透明の実寸プレビュー**が出る。**光った穴をタップ=スナップ接続**、**床をタップ=そのへんに仮置き**(穴ピッチ5mm・タミヤ互換グリッド)。仮置きしたパーツは選択してドラッグで移動、**3軸の90°回転**(水平/前後/左右)・5mm昇降もできる。**ドラッグ中に軸の合う穴へ近づくと吸い付く**(緑のリングが目印)ので、そのまま📌ピンを打てばズレなくつながる。
2. **あとからつなぐ** — ドラッグでスナップして置くと**金色のピンボタン**が出るので、押すだけでその場で結合(ワンクリック)。「📌ピンでとめる」モードでも、**穴が重なっていれば1タップで相手穴を自動選択**して即ピン留め。重なっていない場合だけ2つめの穴をタップ(別のかたまりなら島ごと吸着して結合、**結合前に合わせた向きはそのまま保たれる**。同じかたまりなら追いピン=からくり用)。「🔓はずして自由にする」で削除せずに切り離して組み替えもできる。
3. **穴とピンの文法** — 留めた箇所はデフォルト**ピン2本=固定**。接続の丸いバッジ(または右パネル)をタップすると**ピン1本=くるくる回る受動関節**に切替。**ピン1本でつながったパーツは、3Dビューで直接つかむと関節まわりに回せる**(2Dリンク機構シミュレータ流の直接操作)。サーボの金色リングは**駆動穴**で、そこに付けたパーツはサーボが回す。取付済みパーツの姿勢は3DCADの合致拘束風に「90°回転(ねじり)/反対面に取付/**同一面のまま180°反転**/取付穴の変更」で調整できる。
4. **サーボはマウント付き** — 底面(軸が上向き)と**背面**(軸が横向き)の2つの取付面があり、どの穴で留めても**出力軸が5mmグリッドに乗る**。ホーンの十字穴に別のサーボを直接留めれば直交2軸関節、うらの軸受けピン+コ字金具で両持ち関節も組める。
4. **整える** — 全体重心(バランスのまんなか)と支持多角形(たおれない範囲)が常時表示。緑/黄/赤で安定を確認。アルミ材やおもりで「重さを味方につける」。
5. **パワーボックス** — サーボ・センサの電力コスト合計に応じて必要グレード(S/M/L)が決まる。**搭載しないと書き出せない**。
6. **からくり** — 重なった2つの穴に追いピンを打つと閉ループ(4節リンク等)が組める。「ぶらぶらメーター」が0になれば機構として閉じた証拠。**閉ループの部品を掴んで回すと、拘束ソルバが他の関節を連動させて機構全体が動く**(テオヤンセン機構のような多重ループもOK。回しきれない角度=可動限界では止まる)。サーボのポーズスライダーでも同様に、クランクを回すと機構ごしに連動する。
7. **書き出す** — 「ロボットをおくり出す」で ロボットパッケージ(`*.robopkg.zip`=素のzip)をダウンロード。**全部品が1つにつながっていること**と**パワーボックス搭載**が条件(バラバラの仮置きが残っていると書き出せない)。

## ロボットパッケージ(*.robopkg.zip)

```
mybot.robopkg.zip(素のzip。解凍すれば中のURDF/MJCFを他のシミュレータやROSツールでそのまま使える)
├── robot.urdf          … URDF(閉ループは1箇所カット+警告)
├── robot.mjcf.xml      … MJCF(シミュレータ主経路。閉ループはequality/connect、
│                          アクチュエータは実トルク上限つきposition/velocity)
├── manifest.json       … 操作割当・アクチュエータ性能・センサ搭載・パワーボックス・重心等
└── thumbnail.png       … 3Dビューのサムネイル
```

- 単位は m / kg / rad、Z-up。リンク質量・重心・慣性はカタログ値と形状プリミティブから自動合成。
- 本MVPではメッシュ同梱は行わず、視覚・衝突ともプリミティブ(箱・円筒・球)で出力します。

## 実装構成

| ディレクトリ | 内容 |
|---|---|
| `src/data/parts.json` | パーツカタログ(データ駆動。構造材31種+サーボ6種+センサ3種+パワーボックス3種。サーボは「実在サーボ+マウント」のセット品で、出力軸が5mmグリッドに整合) |
| `src/core/` | UI非依存の純ロジック:穴座標(`holes.ts`)/リンク合成・関節判定・FK(`assembly.ts`)/質量・重心・慣性(`mass.ts`)/安定判定(`stability.ts`)/電力(`power.ts`) |
| `src/core/export/` | URDF・MJCF・manifest・robopkg(zip)エクスポータ |
| `src/state/` | zustandストア(アンドゥ/リドゥ・自動保存) |
| `src/ui/` | React + React Three Fiber のエディタUI(カタログ/3Dビュー/インスペクタ/チュートリアル) |

設計上の要点:

- **内部表現が正**(企画書§5.1)。接続レコードは「どの穴とどの穴を何本のピンで留めたか」だけで、URDF/MJCFはそこからのエクスポータ。
- **pins≧2の接続で結ばれたパーツ群を1リンクに合成**、pins=1は受動関節、駆動穴は能動関節(別紙2§5の変換則そのまま)。
- ゲームバランス(電力コスト・ボックス閾値・質量)はすべてJSONデータ駆動でコード変更なしに調整可能。
