# メモ:テンプレート4種の実装で見えたパーツカタログの過不足

- 作成日:2026-07-23 / **改訂:2026-07-23(§2の提案を実装、§5に「マウント付きサーボ」刷新を追記)**
- 位置づけ:別紙2「フレームパーツカタログ」§8-2(初期セットの過不足はプレイテストで判断)への実装側からのフィードバック。
  ひながた(2輪車/犬型4足/人型2足/虫型6足)を実際に組んで確認した結果。

## 1. 現行セットで組めたもの(良かった点)

- **4種すべて、穴ピン文法だけで成立した。** ホーン・フランジ穴標準装備の効果が大きく、専用マウントなしでサーボ→骨→サーボの連鎖が組める。最小セットの直交性はおおむね達成できている。
- 犬型4足(股+ひざ=マイクロサーボ8個)は電力コスト合計8で**パワーボックスSの容量8ちょうど**。「S箱に収めたくなる誘惑」(企画書§3.3)の見せ場として教材性が高い。
- むきかえ金具(小)+マイクロサーボ+ほね(短)×2+まがりほね+あしうら、の**7点で2自由度の脚モジュール**が組める。これはレシピカード化する価値がある。

## 2. 足りない・噛み合わないもの → **実装済み**

初版で挙げた7件は、企画者レビューを受けてすべてカタログに反映した。

1. ✅ **スペーサ(げた)ブロック** — `SP-05/10/15`(20×20mm、5/10/15mm厚、プラ/アルミ)を追加。
2. ✅ **キャスター全高** — 取付面→球底を**20mm**に改定(タイヤ(大)+背面マウント直付けの軸高と無加工で揃う)。
3. ✅ **マイクロサーボの両持ち** — マウント刷新でマイクロ系にも**軸受けピン(アイドラー穴)**が付き、
   専用のコ字金具 `JT-BRumic` を追加(駆動穴に取付け、反対側は追いピンで軸受けに留める)。
4. ✅ **2軸股関節** — ホーン穴を軸まわり±5mmの**十字グリッド配置**に変更したため、
   ホーンにサーボの底面/背面マウントを直接ピン留めするだけで直交2軸が組める(専用金具は不要になった)。
5. ✅ **パワーボックス上面穴** — S/M/L全てに上面グリッド穴を追加(頭や飾りを積める)。
6. ✅ **前向きの顔** — めだまパネルをドーム同様の**1穴ピン足**に変更(くるくる回して向きを決める)。
7. ✅ **ほね中間長** — `FR-B045` / `FR-B075` を追加。

## 3. サーボの「マウント付きモデル」刷新(2026-07-23)

**課題:** 実在サーボをそのまま5mmグリッドに載せると、取付穴と出力軸の位置関係が中途半端で
(SG90は取付穴間28mm・軸オフセット5.5mmなど)、軸がグリッドから外れて扱いにくい。

**方針:** 実在部品コンセプトは守り、**「実在サーボ+樹脂マウント」のセット品**としてモデル化した
(実機でも市販マウントに入れて使うのが普通)。

- **出力軸をパーツ原点に固定。** マウントの全取付穴は軸から5mmグリッド整合の位置にあり、
  どの穴で留めても軸がグリッドに乗る。
- **取付面は2つ:** 底面(軸は面に垂直=ヨー向き)と**背面**(軸は面に平行=水平向き)。
  背面マウントのおかげで、**むきかえ金具なしで軸を横向きにできる**
  (旧テンプレの「金具+サーボ」2部品が1部品になり、2輪車は13→8部品に減った)。
- **ホーン穴はリング(半径7.5mm)をやめて十字(軸±5mm)に。** ホーンに留めた骨・板・サーボのすべてが
  グリッド整合になる。
- **軸受けピン(アイドラー)を全サーボに装備**(マウント側の機能として)。両持ち関節が全サイズで組める。
- フランジやサーボ本体の寸法・色は実物準拠のまま(見た目のリアリティ維持)。

## 4. テンプレ実装者向けの寸法メモ(ハマりどころ)

- **左右対称は「鏡像の取付穴」で作る。** 回転だけでは鏡像にできないため、左側のサーボは本体の寝る向きが
  反転する。背面マウントの取付穴を左右で鏡像に選ぶ(x=-5の穴↔x=+5の穴)と軸線が左右でそろう。
- 背面マウントで水平置きしたときの**軸高は取付面+11.5mm**(穴面3mm+軸まで8.5mm)。
  タイヤ(大)r32.5と新キャスター(20mm)がほぼ揃う組み合わせになっている。
- ひざサーボは**本体を下向き**に付ける(上向きだと、すねが太ももより上から始まり、ほねの先端が足裏より下に突き出る)。
- 6足の前後45°脚は、まっすぐ下の中脚と接地高さを合わせるため、**中脚だけ足首の取付穴を近づける**(60mm脚:45°→穴10、垂直→穴7でズレ0.4mm)。

## 5. リンク機構の連動操作(2026-07-23追記)

閉ループ(からくり)を含む機構の直接操作に、数値拘束ソルバ(ガウス・ニュートン+連続化)を導入した
(`src/core/linkage.ts`)。linksimのCanMove(拘束伝播)に相当する機能の数値版。

- **掴んで回す**:掴んだ受動関節の角度を入力に、同じ機構内の他の受動関節を
  「ループピンの穴間差ベクトルがrest時から変わらない」拘束を保つように解く(rest姿勢を更新)。
  テオヤンセン機構のような多重ループにもそのまま対応する。
- **ポーズスライダー**:サーボ角を固定入力に受動関節の表示角を解く(表示のみ)。
  クランク駆動のからくりがスライダーで連動して動く。
- 実装メモ:残差は穴間距離(ノルム)ではなく**差ベクトルの3成分**にする(ノルムは解の点で微分不能で振動する)。
  大きな角度変化は8°刻みの連続化+**前ステップの速度外挿**で、特異姿勢(リンク一直線)を通過しても
  連続な運動の枝を追跡する。可動限界(拘束誤差3mm超)では動かさない。
- 制限:拘束は平面機構を想定(ピン軸が運動中一定)。立体的に軸が回る閉ループは今後の課題。

## 6. ヤンセン8足テンプレート(2026-07-23追記)

テオヤンセン機構×4ユニット(各クランクピンに前後鏡像2脚)・モータ2つの8足歩行機
「ヤンセンの8ほんあし」を追加した。設計メモ:

- **リンク長はホーリーナンバーの5mmグリッド再設計**。原寸の量子化(丸め)では特異マージンが
  1.2mmしかなくソルバが位相通過で破綻するため、2D円交点法の全回転掃引を評価関数に
  総当たり探索し、**マージン7.7mm・歩幅51mm・リフト13mm**の組を採用:
  `a40 l15 m15 b50 c40 d35 e55 f40 g40 h70 i50 j55 k60`
  (すべて ほね短め/中/中長 の穴間距離で実現)。
- **組立は数値ベース**:2Dソルバで節点座標→各ほねを方向指定(2°刻み探索)で仮置き→
  ループピンを張って `settleLoops` で厳密に閉じる(誤差0.5mm未満を保証)。
- **クランクはダブル**:ほね(中長)のまんなかの穴を駆動穴に直接固定し、両端±15mmを
  クランクピンにする(位相0°/180°)。rest角は4位相の特異マージンが最大になる90°。
- 開発で踏んだ穴:表示系の全域木BFSが追いピン経由で先に張られると、駆動エッジが
  ループ扱いになりFKが壊れる。**全域木はtree接続+能動関節を優先**して張り、
  ループ拘束は「木に入らなかった関節すべて」を対象にするよう修正した(`assembly.ts`/`linkage.ts`)。
